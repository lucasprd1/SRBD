Módulo "DuplicateKey"

Este módulo foi desenvolvido para identificar, remover e corrigir registros duplicados em tabelas do PostgreSQL. Ele opera com base em uma coluna de identificação (ID) que você informa.

O módulo oferece três funções principais:

- Verificar Duplicatas: Realiza uma consulta para contar quantos registros duplicados existem na tabela. Ele usa a coluna ID para agrupar as linhas e identificar as cópias.

- Ajustar Tabela: Deleta todos os registros duplicados encontrados, mantendo apenas a primeira ocorrência. Essa ação é feita para garantir que você tenha apenas uma linha para cada valor único de ID.

- Ajustar Sequence: Corrige a sequência de IDs da tabela, garantindo que o próximo ID gerado seja o maior valor atual, mais um. Isso evita erros ao tentar inserir novos registros após a exclusão de duplicatas.

Para usar, basta informar o nome da tabela (schema.table) e da coluna ID, e então executar a ação desejada.

Módulo "Could Not Read..."

Este módulo foi projetado e desenvolvido para identificar e excluir dados corrompidos em colunas dos tipos text, varchar e character varying em tabelas do PostgreSQL. A ferramenta utiliza uma função personalizada chamada chk, que é criada diretamente no banco de dados.

O módulo oferece quatro funções principais:

- Criar Função: Adiciona a função chk ao seu banco de dados. Essa função é usada para identificar registros corrompidos que não conseguem ser processados corretamente.

- Verificar Linhas: Conta e exibe o número de registros corrompidos na tabela e coluna especificada com base na função criada (a verificação não irá funcionar caso não exista a função "chk").

- Ajustar Tabela: Exclui permanentemente todos os registros corrompidos encontrados (a exclusão não irá funcionar caso não exista a função "chk").

- Remover Função: Apaga a função chk do banco de dados quando ela não for mais necessária para você.

Para usar, basta informar o nome da tabela (schema.table) e da coluna, e então executar a ação desejada.

Módulo "Otimização Completa"

Este módulo foi desenvolvido para ajudar a otimizar o desempenho e a saúde do seu banco de dados PostgreSQL. Ele consolida as principais ferramentas de manutenção do PostgreSQL em um único local, permitindo que você execute operações de forma manual e segura.

O módulo oferece as seguintes funções:

- Otimizar PostgreSQL: Ajusta as configurações do servidor (postgresql.conf) para melhorar o desempenho. Ele calcula valores ideais para parâmetros-chave (como shared_buffers e work_mem) com base na RAM, CPU e no número de estações de trabalho que você fornecer. É importante deixar claro que para a otimização surtir efeito, o PostgreSQL precisa ser reiniciado.

- ANALYZE: Analisa todas as tabelas do banco de dados e atualiza as estatísticas do otimizador de consultas. Isso ajuda o PostgreSQL a tomar melhores decisões sobre como executar as queries, resultando em um desempenho mais rápido.

- REINDEX: Reconstrói todos os índices do banco de dados. Essa operação é crucial para corrigir índices corrompidos e para reorganizar os dados, o que pode melhorar a velocidade de consulta.

- VACUUM FULL: Reorganiza as tabelas do banco de dados, recuperando espaço em disco e eliminando fragmentação. Essa é uma operação mais pesada que o VACUUM normal e pode levar mais tempo, mas é eficaz para liberar espaço de armazenamento não utilizado.

Para usar, basta preencher os campos com as informações do seu servidor (Versão do PostgreSQL, RAM, CPU e número de estações) e, em seguida, escolher a operação desejada.

Módulo "Otimização por Schema"

Este módulo foi desenvolvido para permitir a otimização de tabelas e índices dentro de um schema específico do PostgreSQL, em vez de otimizar o banco de dados inteiro. Isso é ideal para ambientes onde diferentes schemas são usados para diferentes aplicações ou clientes.

O módulo oferece as seguintes funções:

- ANALYZE Schema: Executa uma análise em todas as tabelas dentro do schema que você especificar. Essa operação ajuda a garantir que o otimizador de consultas do PostgreSQL tenha estatísticas atualizadas para gerar planos de execução mais eficientes, melhorando o desempenho geral das consultas.

- REINDEX Schema: Reconstrói todos os índices dentro do schema. O REINDEX é útil para corrigir índices corrompidos ou reorganizar índices que se tornaram ineficientes com o tempo, o que pode acelerar as consultas que dependem desses índices.

Para usar, basta informar o nome do schema desejado e, em seguida, escolher a operação.

Módulo "Uncommitted xmin"

Este módulo foi projetado para resolver um problema específico de bancos de dados PostgreSQL: a visibilidade de registros que ficaram "congelados" devido a um ID de transação antigo (conhecido como Xmin). Isso acontece em bancos de dados que são replicados ou em que o VACUUM não é executado de forma eficiente.

A ferramenta faz o seguinte:

- Sincronização: Executa uma atualização em uma tabela e coluna específicas. Embora a atualização não mude os dados visíveis, ela força o PostgreSQL a marcar as linhas como modificadas na transação atual, resolvendo o problema de visibilidade de dados.

Para usar, você deve informar o nome completo da tabela no formato schema.tabela e o nome de uma coluna para que a operação possa ser executada.
